Features

Everything you need for multi-repo development.

Stop juggling branches across repos. Create a workspace, do your work, tear it down.

Instant workspaces

One command creates worktrees for every repo in your group, with a VS Code workspace file ready to open.

Fully parallel

Clones, syncs, pulls, pushes, and hooks run concurrently across all repos. Built in Go.

Short repo names

Set your git org once. Use my-app instead of the full URL. GitHub, GitLab, Bitbucket, or self-hosted.

Lifecycle hooks

Auto-run pnpm install or any setup command after project creation. Global, group, or per-repo.

Context-aware

hk pull, hk push, hk status auto-detect your project from the current directory.

Self-updating

Checks for updates daily. hk update upgrades in place — no sudo after initial install.

Quick Start

Up and running in under a minute.

Install, create a config, sync your repos, and start a new workspace. Both hike and hk are installed automatically.

~/projects
$ hk init
Created hike.yaml

$ hk sync
Syncing group "platform" (3 repos)...
  my-app: cloned
  shared-lib: cloned
  api: cloned

$ hk platform
Creating "platform-bold-cedar"...
Project "platform-bold-cedar" created.

$ cd platform-bold-cedar
$ hk status
  my-app       branch: platform-bold-cedar
  shared-lib   branch: platform-bold-cedar
  api          branch: platform-bold-cedar
● ● ● hike.yaml
git:
  org: your-org

hooks:
  onCreate: pnpm install

groups:
  platform:
    default: true
    repos:
      - my-app
      - shared-lib
      - repo: api
        branch: develop

  marketing:
    aliases: [mktg]
    repos:
      - website
      - cms
Configuration

One YAML file.
That's it.

Define your git org, groups, repos, and hooks. Short names expand automatically. Override anything per-repo.

Commands

Simple, memorable commands.

Every command works with hike or hk. Most commands auto-detect the current project when run from inside one.

hk [group] [name] default

Create a new project. The first argument is matched against known groups. If it matches, it's the group; otherwise it's the project name using the default group. Omit the name to generate a random one.

$ hk platform my-feature # platform-my-feature/ $ hk platform # platform-bold-cedar/ (random) $ hk platform -c purple # with VS Code title color $ hk platform -c # random color
hk init

Create a new hike.yaml config file in the current directory with an annotated example. Prompts before overwriting.

hk sync [-g group]

Clone any missing repos and hard-reset all .hike/ repos to the latest origin/HEAD. This is the command to run after adding new repos to your config. Runs all clones in parallel.

Performs git reset --hard on .hike/ repos. Never work directly in .hike/ — use project worktrees instead.
hk pull [project]

Run git pull --ff-only on every repo in the project. Auto-detects the project from cwd, or pass a name explicitly.

hk push [project]

Run git push on every repo in the project. Runs in parallel across all repos.

hk status [project]

Show the branch, dirty state, and ahead/behind status for each repo in the project.

$ hk status my-app branch: platform-bold-cedar shared-lib branch: platform-bold-cedar [dirty] api branch: platform-bold-cedar
hk delete [project]

Remove a project directory and clean up all its git worktrees and branches. Auto-detects the project from cwd.

hk list

List all projects in the current hike root.

hk update

Download and install the latest release. Binary lives in ~/.hike/bin/ so no sudo is needed after the initial install.
